# 深度分析方案对比报告

> 日期: 2026-03-20 17:36
> 分析工具: Claude Code (glm-5)

---

## 一、方案概述

### 完整模式
- **内容来源**: PDF 全文 (最多 60000 字符)
- **AI 调用**: kimi-k2.5 (报告) + kimi-k2.5 (JSON提取)
- **max_tokens**: 默认值

### 快速模式
- **内容来源**: 仅摘要 (~500 字符)
- **AI 调用**: kimi-k2.5 (报告, 4000 tokens) + glm-5 (JSON提取)
- **max_tokens**: 4000 (报告生成)

---

## 二、性能对比

| 指标 | 完整模式 | 快速模式 | 差异 |
|------|----------|----------|------|
| 单篇耗时 | ~285 秒 | ~142 秒 | **快 50%** |
| 处理速度 | ~7 篇/小时 | ~21 篇/小时 | **3倍** |
| 300 篇预估 | ~42 小时 | ~14 小时 | **节省 28 小时** |
| 报告行数 | ~543 行 | ~346 行 | 少 36% |
| 报告大小 | ~25KB | ~17KB | 少 32% |

## 三、质量对比

### 3.1 报告结构

两种模式的报告结构基本一致：
- ✅ 基础信息
- ✅ 一句话总结
- ✅ 论文大纲
- ✅ 研究背景与动机
- ✅ 相关工作概览
- ✅ 核心方法详解
- ✅ 实验设计与验证
- ✅ 可解释性分析
- ✅ 行动建议
- ✅ 知识关联

### 3.2 内容深度差异

| 方面 | 完整模式 | 快速模式 | 影响 |
|------|----------|----------|------|
| **实验数据** | 真实数据 (如 PSNR 29.25 dB) | 推断数据 | ⚠️ 可能不准确 |
| **消融实验** | 详细表格 + 具体数值 | 简化或缺失 | ⚠️ 深度不足 |
| **公式推导** | 完整数学建模 | 有公式但可能推断 | ⚠️ 可能简化 |
| **算法伪代码** | 完整流程 | 有但可能简化 | - |
| **可视化分析** | 基于实际图表 | 推断性描述 | ⚠️ 可能不准确 |

### 3.3 样例对比

#### 完整模式 - Matryoshka Gaussian Splatting

```
实验结果:
| 方法 | Mip-NeRF 360 PSNR | 全容量退化 | 连续LoD |
| 3DGS | 29.23 dB | - | ✗ |
| MGS (Ours) | 29.25 dB | 无 | ✓ |

消融实验:
| 排序策略 | 全容量PSNR | 低预算PSNR |
| 可学习排序键 | 29.25 | 26.2 |

特点: 数据来自 PDF 实际内容，有具体数值
```

#### 快速模式 - Spectrally-Guided Diffusion

```
实验结果:
| 方法 | FID↓ | sFID↓ |
| EDM | 2.04 | 4.85 |
| EDM + Ours | 1.89 | 4.52 |

特点: 数据可能是 AI 根据摘要推断的
```

### 3.4 关键发现

**快速模式的局限**：
1. **实验数据来源不明** - 可能是 AI 根据"应该有的结果"推断
2. **消融实验简化** - 缺少真实论文中的详细消融分析
3. **可视化描述模糊** - 出现"论文应包含..."表述

**完整模式的优势**：
1. **数据真实可靠** - 直接从 PDF 提取
2. **消融实验详细** - 有完整的实验设置和结果
3. **公式推导准确** - 基于实际论文内容

## 四、建议

### 4.1 场景选择

| 场景 | 推荐模式 | 原因 |
|------|----------|------|
| **快速筛选** | 快速模式 | 速度优先，获取概览 |
| **深度学习** | 完整模式 | 需要准确实验数据 |
| **重要论文** | 完整模式 | 数据准确性重要 |
| **批量处理** | 快速模式 | 效率优先 |

### 4.2 混合策略建议

```
论文重要性 = f(Tier, 领域相关性, 个人兴趣)

if Tier == "A" 或 领域高度相关:
    使用完整模式  # 确保数据准确
else:
    使用快速模式  # 效率优先
```
