# 论文 Tier 动态评估规则

> 版本: 2.1
> 最后更新: 2026-03-31

## 核心原则

Tier 是**动态**的，随论文生命周期变化：
- **新论文**：基于内容质量评分
- **成熟论文**：结合引用数和时效性
- **高分论文保护**：评分>=85的论文有保底机制

## Tier 定义

| Tier | 含义 | 特征 |
|------|------|------|
| A | 顶尖论文 | 高质量内容 + 高关注度（引用/讨论） |
| B | 有价值论文 | 质量不错，值得参考 |
| C | 一般参考 | 普通论文，特定场景参考 |

## 分阶段评估

### 阶段1：入库时（新论文）
**纯内容评分**，不考虑引用数（因为新论文没有引用）

```
分数 = 方向热度(40) + 机构权重(15) + 主题相关性(20) + 创新信号(15) + 关键词加分(10)

Tier A: 分数 >= 80
Tier B: 分数 50-79
Tier C: 分数 < 50
```

### 阶段2：成熟期（发布 30 天后）
**内容评分 + 引用数加成 + 时效性衰减 + 保底机制**

```
最终分数 = 内容评分 × 时效系数 + 引用加成

时效系数：
- 0-30天: 1.0（新论文红利）
- 31-90天: 0.95
- 91-180天: 0.90
- 180天以上: 0.85

引用加成：
- 引用 >= 100: +20
- 引用 >= 50: +10
- 引用 >= 20: +5
- 引用 >= 10: +2
```

### 阶段3：保底机制（v2.1新增）

**高分论文保护**：评分>=85的论文有特殊保护：
- 默认保持 Tier A
- 仅在发布>365天且引用=0时，才降级到 Tier B

**高引用论文保护**：
- 引用>=50且评分>=40的论文，至少 Tier B

## 实现建议

### 1. 初始 Tier（入库时）
```python
def get_initial_tier(score: int) -> str:
    if score >= 80: return 'A'
    elif score >= 50: return 'B'
    else: return 'C'
```

### 2. 动态 Tier（定期更新）
```python

    score: int,
    citation_count: int,
    days_since_publish: int
) -> str:
    # 新论文保持初始评估
    if days_since_publish <= 30:
        return get_initial_tier(score)
```

## 示例

### 示例1：新论文（刚入库）
- 内容评分：85
- 引用数：0
- 发布天数：1
- **Tier: A**（内容评分 85 >= 80）

### 示例2：高分论文（DeepSeek-V3）
- 内容评分：88
- 引用数：0
- 发布天数：321
- **Tier: A**（保底机制：评分>=85）

### 示例3：成熟论文（6个月后）
- 内容评分：75
- 引用数：150
- 发布天数：180
- 时效系数：0.90
- 引用加成：+20
- 最终分数：75 × 0.90 + 20 = 87.5
- **Tier: A**（引用加成提升）

### 示例4：内容高分但无引用
- 内容评分：82
- 引用数：0
- 发布天数：200
- 时效系数：0.85
- 最终分数：82 × 0.85 = 69.7
- **Tier: B**（从 A 降级，但低于85分保底线）

## 注意事项

1. **Tier 会变化** - 这是正常的，反映论文真实价值
2. **新论文红利** - 前30天保持初始评估
3. **高分论文保护** - 评分>=85的论文不容易降级
4. **引用是重要信号** - 但不是唯一标准
5. **内容为王** - 高质量内容即使无引用也至少是 Tier B
